# 自动化运维平台 OpsCenter — 安装说明

## 系统要求

| 组件 | 要求 |
|---|---|
| Node.js | v18 或以上（推荐 v20/v22） |
| pnpm | v8 或以上（`npm install -g pnpm`） |
| PostgreSQL | v14 或以上 |
| 操作系统 | Linux（VPS 推荐 Ubuntu 22.04） |

---

## 一、VPS 服务端部署

### 1. 安装依赖

```bash
# 安装 Node.js (Ubuntu)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 pnpm
npm install -g pnpm

# 安装 PostgreSQL
sudo apt-get install -y postgresql postgresql-contrib
```

### 2. 创建数据库

```bash
sudo -u postgres psql

# 在 psql 中执行：
CREATE DATABASE opscenter;
CREATE USER opscenter WITH ENCRYPTED PASSWORD 'your_strong_password';
GRANT ALL PRIVILEGES ON DATABASE opscenter TO opscenter;
\q
```

### 3. 配置环境变量

在项目根目录创建 `.env` 文件（或直接 export 到 shell）：

```bash
export DATABASE_URL="postgresql://opscenter:your_strong_password@localhost:5432/opscenter"
export SESSION_SECRET="your_very_long_random_secret_string_here"
export PORT=8080           # API 服务端口
export BASE_PATH=/         # 前端基础路径
```

也可以在 `lib/db/drizzle.config.ts` 中直接写死 DATABASE_URL（不推荐）。

### 4. 安装依赖并初始化数据库

```bash
# 在项目根目录
pnpm install

# 推送数据库 Schema（创建所有表）
pnpm --filter @workspace/db run push
```

### 5. 构建生产版本

```bash
# 构建 API 服务器
pnpm --filter @workspace/api-server run build

# 构建前端（静态文件）
pnpm --filter @workspace/dashboard run build
```

### 6. 配置 Nginx 反向代理（推荐）

```nginx
server {
    listen 80;
    server_name your-vps-domain.com;
    
    # 前端静态文件
    location / {
        root /path/to/project/artifacts/dashboard/dist/public;
        try_files $uri $uri/ /index.html;
    }
    
    # API 和 WebSocket
    location /api {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }
}
```

启用 HTTPS（推荐使用 Certbot）：
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-vps-domain.com
```

### 7. 启动 API 服务

```bash
# 直接启动（测试用）
DATABASE_URL="..." SESSION_SECRET="..." PORT=8080 \
  node artifacts/api-server/dist/index.mjs

# 使用 PM2 守护进程（推荐生产环境）
npm install -g pm2
pm2 start artifacts/api-server/dist/index.mjs \
  --name opscenter-api \
  --env DATABASE_URL="..." \
  --env SESSION_SECRET="..." \
  --env PORT=8080

pm2 save
pm2 startup
```

### 8. 验证部署

打开浏览器访问 `http://your-vps-domain.com`

- 默认账号：`admin`
- 默认密码：`admin123`
- **首次登录后请立即在数据库或后台修改密码！**

## 二、Agent 编译（Windows EXE）

Agent 源码在 `artifacts/agent/` 目录。

### 在 Windows 机器或有 Node.js 的环境中编译：

```bash
cd artifacts/agent

# 安装依赖
npm install

# 编译 TypeScript
npx tsc

# 打包为单个 EXE（Windows 64位）
npx pkg dist/index.js --targets node18-win-x64 --output dist/ops-agent.exe
```

编译完成后，`dist/ops-agent.exe` 就是可以分发的单文件。

### 其他平台编译目标：

```bash
# macOS
npx pkg dist/index.js --targets node18-macos-x64 --output dist/ops-agent-mac

# Linux
npx pkg dist/index.js --targets node18-linux-x64 --output dist/ops-agent-linux
```

## 三、Agent 安装到内网机器

### 首次安装

将 `ops-agent.exe` 复制到目标机器，然后运行：

```cmd
ops-agent.exe --server wss://your-vps-domain.com --key YOUR_AGENT_KEY --name "机器描述名称"
```

参数说明：
- `--server`：VPS 地址，使用 `wss://`（HTTPS 环境）或 `ws://`（HTTP 环境）
- `--key`：Agent 密钥（任意字符串，每台机器建议用不同的密钥）
- `--name`：机器在后台显示的名称

**首次运行会弹出提醒窗口**，告知用户这是自动化运维软件，点击 OK 后程序静默运行。
之后每次启动都不再弹窗。

### 设置开机自启（Windows）

**方法一：放入启动文件夹**
```
Win + R → 输入 shell:startup → 将 ops-agent.exe 快捷方式放入此文件夹
```

**方法二：注册为 Windows 服务（推荐）**
```cmd
# 使用 NSSM（Non-Sucking Service Manager）
# 下载 nssm: https://nssm.cc/download

nssm install OpsAgent "C:\Path\To\ops-agent.exe"
nssm set OpsAgent AppParameters "--server wss://your-vps.com --key YOUR_KEY"
nssm start OpsAgent
```

### 配置文件

Agent 运行后会在用户主目录保存配置：
- Windows: `C:\Users\<用户名>\.ops-agent.json`
- macOS/Linux: `~/.ops-agent.json`

内容示例：
```json
{
  "serverUrl": "wss://your-vps-domain.com",
  "agentKey": "pc-office-01-secret-key",
  "name": "办公室-PC-01"
}
```

之后直接运行 `ops-agent.exe` 无需重复参数。

## 六、安全建议

1. **立即修改默认密码** — 直接在 PostgreSQL 中更新 `users` 表
2. **使用 HTTPS** — 配置 Certbot/Let's Encrypt，Agent 连接用 `wss://`
3. **防火墙** — 仅开放 80/443 端口，8080 端口不对外
4. **Agent 密钥** — 每台机器使用不同的 Key，方便单独吊销
5. **定期查看审计日志** — 后台 Audit Logs 页面记录所有操作
